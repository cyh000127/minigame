# 개발 가이드

이 문서는 미니게임을 독립적으로 추가하고 관리하기 위한 기준입니다.

## 목표

- 게임별 코드가 서로 섞이지 않게 유지한다.
- 새 게임을 만들 때 기존 게임 구현에 영향을 받지 않는다.
- 모든 게임 구현 코드를 TypeScript 기반으로 관리한다.
- 게임 하나를 삭제하거나 교체해도 다른 게임이 깨지지 않게 한다.

## 디렉토리 규칙

모든 게임은 `games/<game-name>/` 아래에 둡니다.

```text
games/
  sample-game/
    README.md
    package.json
    src/
    assets/
    tests/
```

각 게임 디렉토리는 다음 내용을 포함할 수 있습니다.

- `README.md`: 게임 설명, 실행 방법, 개발 메모
- `package.json`: 게임별 실행, 빌드, 테스트 스크립트
- `src/`: 게임 소스 코드
- `assets/`: 해당 게임에서만 사용하는 이미지, 사운드, 폰트 등
- `tests/`: 해당 게임 전용 테스트
- 게임별 설정 파일: 빌드 도구, 패키지 설정, 린트 설정 등

## TypeScript 규칙

- 게임 구현 코드는 TypeScript 또는 TSX로 작성한다.
- 타입 정의는 가능한 한 해당 게임 디렉토리 안에서 관리한다.
- 서버, 클라이언트, 공유 엔진이 필요하면 대상 게임의 `src/` 아래에서 역할별로 나눈다.
- 외부 라이브러리 설정은 게임별 `package.json`과 설정 파일에 둔다.

## 독립성 규칙

각 게임은 다른 게임의 코드를 보거나 참조하지 않는 것을 원칙으로 합니다.

- 다른 게임 디렉토리에서 import하지 않는다.
- 다른 게임의 파일을 복사해서 시작하지 않는다.
- 다른 게임의 에셋을 재사용하지 않는다.
- 루트에 `shared`, `common`, `utils` 같은 공유 코드 디렉토리를 만들지 않는다.
- 게임 간 동작 방식이 비슷해도 각 게임 안에서 독립적으로 구현한다.

## 작업 순서

1. `docs/`와 루트 문서를 읽는다.
2. 새 게임 이름을 정하고 `games/<game-name>/` 디렉토리를 만든다.
3. 해당 게임 디렉토리 안에서만 소스, 에셋, 테스트, 설정을 작성한다.
4. 게임별 `README.md`에 실행 방법과 주요 결정을 기록한다.
5. 루트 실행기에서 선택할 수 있도록 `src/launcher.ts`의 `GAMES` 배열에 메타데이터를 추가한다.
6. 변경 사항을 테스트한 뒤 커밋 컨벤션에 맞춰 커밋한다.

## 루트 실행기 연동

루트 실행기는 게임 코드를 import하지 않고 `games/<game-name>/index.html`을 iframe으로 로드합니다.

- 게임의 `index.html`은 엔트리 스크립트를 `./src/main.ts`처럼 상대 경로로 작성한다.
- 새 게임의 제목, 장르, 설명, 조작키, 실행 상태는 루트 `src/launcher.ts`의 `GAMES` 배열에 추가한다.
- `start()`, `pause()`, `gameOver()` lifecycle 계약이 필요하면 해당 게임 내부에서만 메시지 어댑터를 구현한다.
- 자세한 기준은 [미니게임 허브 연동 가이드](minigame-hub.md)를 따른다.

## 예외 처리

루트 설정 변경이 꼭 필요한 경우에는 다음 기준을 지킵니다.

- 변경 범위가 모든 게임에 영향을 주는지 확인한다.
- 특정 게임에만 필요한 설정이면 해당 게임 디렉토리에 둔다.
- 루트 설정 변경 이유를 커밋 메시지나 관련 문서에 남긴다.
