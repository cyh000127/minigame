# 커밋 컨벤션

커밋 메시지는 `prefix: 한글 설명` 형식을 사용합니다.

```text
prefix: 무엇을 변경했는지 한글로 요약
```

## 기본 규칙

- prefix는 영어 소문자로 작성한다.
- 콜론 뒤에 한 칸을 띄운다.
- 설명은 한글로 작성한다.
- 설명은 명령형보다 변경 내용을 짧고 분명하게 적는다.

## Prefix 목록

| Prefix | 사용 시점 |
| --- | --- |
| `feat` | 새로운 게임이나 기능을 추가할 때 |
| `fix` | 버그를 수정할 때 |
| `docs` | 문서만 변경할 때 |
| `style` | 포맷팅, 세미콜론, 공백 등 동작 변화 없는 스타일 변경 |
| `refactor` | 기능 변화 없이 코드 구조를 개선할 때 |
| `test` | 테스트를 추가하거나 수정할 때 |
| `chore` | 빌드, 설정, 패키지 관리 등 보조 작업 |
| `assets` | 이미지, 사운드, 폰트 등 에셋을 추가하거나 수정할 때 |

## 예시

```text
feat: 숫자 맞추기 게임 추가
fix: 카드 뒤집기 점수 계산 오류 수정
docs: 게임 독립 개발 가이드 추가
style: 테트리스 코드 포맷 정리
refactor: 뱀 게임 입력 처리 구조 개선
test: 두더지 잡기 타이머 테스트 추가
chore: 루트 gitignore 설정 추가
assets: 벽돌깨기 효과음 추가
```

## 게임별 커밋 권장 방식

- 한 커밋에는 가능하면 한 게임의 변경만 담는다.
- 여러 게임을 동시에 수정하지 않는다.
- 문서 변경과 게임 구현 변경은 가능한 한 분리한다.
- 루트 설정을 바꾸는 커밋은 영향 범위를 커밋 본문에 적는다.
